Trabalhando em conjunto

  • O que é versionamento de código?

Versionamento é como tirar “fotos” do seu projeto ao longo do tempo. Ele permite que você volte atrás, experimente sem medo e acompanhe cada alteração feita no código.

Git e GitHub: a dupla essencial

Git é o sistema de versionamento mais popular do mundo. Ele roda no seu computador e gerencia o histórico local do projeto. Com ele, você cria versões (chamadas de commits), volta no tempo, cria ramificações (branches) e junta alterações.

GitHub é uma plataforma online que armazena repositórios Git na nuvem. Ele facilita o trabalho em equipe, permitindo que várias pessoas colaborem no mesmo projeto, revisem código e acompanhem tarefas.

Fluxo básico

  • git init – começa o versionamento no seu projeto
  • git add – prepara as mudanças
  • git commit – salva uma versão com descrição
  • git push – envia as versões para o GitHub
  • git pull – traz atualizações do GitHub para o seu computador

Por que aprender?

  1. Segurança: nunca mais perca um código importante
  2. Colaboração: trabalhe com outras pessoas sem sobrescrever o trabalho delas
  3. Portfólio: o GitHub é o currículo de quem programa

Instalando o Git

O Git é o sistema de controle de versão mais popular do mundo. Ele é um software de código aberto que permite aos desenvolvedores rastrear mudanças em seus projetos, colaborar com outros e gerenciar diferentes versões do código. O Git é amplamente utilizado em projetos de software, mas também pode ser aplicado a qualquer tipo de projeto que envolva arquivos e mudanças frequentes.

Para instalar o Git, siga as instruções abaixo de acordo com o seu sistema operacional: - Windows: Baixe o instalador do Git para Windows em https://git-scm.com/download/win e siga as instruções de instalação.

  • Mac
    • Se você tiver o Homebrew instalado, pode usar o comando no Terminal: brew install git
    • Alternativamente, você pode baixar o instalador do Git para Mac em https://git-scm.com/download/mac e seguir as instruções de instalação.
  • Linux: Use o gerenciador de pacotes da sua distribuição para instalar o Git. Por exemplo, em distribuições baseadas no Debian/Ubuntu, você pode usar: sudo apt-get update sudo apt-get install git Após a instalação, você pode verificar se o Git foi instalado corretamente abrindo um terminal e digitando: git --version

Isso deve retornar a versão do Git instalada no seu sistema.

Configurando o Git no RStudio

Agora, vamos ao RStudio para configurar o acesso do Git. No RStudio ache a aba Terminal e insira os seguintes comados, um de cada vez.

Substitua USER e USER@SEUEMAIL.COM com o seu. No meu caso seria cammcordeiro e cammcordeiro@pq.uenf.br.

# mostrando a versão do git em seu computador
git --version

# mostrando o caminho das pastas em seu Mac/Linux
which git

# mostrando o caminho das pastas no Windows
where git

# troque USUARIO com o seu nome de usuário do Github
git config –-global user.name USUARIO

# troque USUARIO@SEUEMAIL com o seu email resgistrado no Github
git config –-global user.email USUARIO@SEUEMAIL.COM

# listando "config" para confirmar as variáveis de user.* 
git config --list

# use git "merge" (default) para resolver possíveis conflitos git "pull" conflicts
git config --global pull.rebase true



Agora acesse Tools -> Global Options e selecione Git/SVN no menu lateral. No campo Git executable insira o caminho das pastas baseados nos comandos anteriores (Note que eles são diferentes para Mac/Linux which git e Windows where git).

Configurando uma chave SSH (Dentro do RStudio)

Configurando uma chave SSH (Fora do RStudio)

SSH (Secure Shell) é um protocolo seguro de comunicação. Ao configurar uma chave SSH no GitHub, você evita digitar suas credenciais toda vez que interagir com o repositório (ex. fazer push ou pull).

Para configurar uma chave SSH, siga as instruções detalhadas em GitHub Docs.

Este processo envolve a geração de uma chave SSH em seu computador e a adição dessa chave à sua conta do GitHub.

Passo 1: Verificar se você já tem uma chave SSH

Abra o terminal (ou Git Bash no Windows) e digite:

ls -al ~/.ssh

Procure por arquivos chamados id_ed25519.pub ou id_rsa.pub. Se existirem, você já tem uma chave. Pule para o Passo 3.

Passo 2: Gerar uma nova chave SSH

No terminal, digite (substitua o e-mail pelo seu):

O terminal vai perguntar:

ssh-keygen -t ed25519 -C "seu-email@exemplo.com"

  • Nota: se o sistema não suportar ed25519, use ssh-keygen -t rsa -b 4096 -C "seu-email@exemplo.com"

  • Onde salvar a chave? Aperte Enter para aceitar o local padrão.

  • Senha? É opcional, mas recomendado. Se colocar, vai precisar digitá-la ao usar a chave. Pronto! A chave foi gerada.

Passo 3: Adicionar a chave ao ssh-agent

  • Inicie o ssh-agent:

eval "$(ssh-agent -s)"

*Adicione a chave privada:

ssh-add ~/.ssh/id_ed25519

Passo 4: Copiar a chave pública

A chave pública é a que você vai colar no GitHub. Para copiar:

Linux/Mac:

cat ~/.ssh/id_ed25519.pub

Selecione e copie a saída (começa com ssh-ed25519).

Windows (Git Bash):

clip < ~/.ssh/id_ed25519.pub

Já copia direto para a área de transferência.

Passo 5: Adicionar a chave no GitHub

  • Acesse github.com/settings/keys (logado no GitHub)
  • Clique em “New SSH Key”
    • Título: um nome descritivo (ex: “Meu notebook pessoal”)
  • Tipo de chave: Authentication Key
  • Chave: cole a chave pública copiada
  • Clique em “Add SSH Key”

Passo 6: Testar a conexão

No terminal, digite:

ssh -T git@github.com

Você verá uma mensagem como: Hi seu-usuario! You’ve successfully authenticated…

Se pedir confirmação de fingerprint, digite yes.

Passo 7: Usar SSH nos seus repositórios

Agora, ao clonar ou configurar um repositório, use a URL SSH:

bash git clone git@github.com:usuario/repositorio.git

Para converter um repositório HTTPS para SSH:

$bash git remote set-url origin git@github.com:usuario/repositorio.git

Essa configuração parece ser desafiadora em um primeiro momento, mas felizmente este procedimento é bem documentado em Happy Git and GitHub for the useR se tiver algum problema.

Colaboração: fork, branch, pull request e merge

Agora que já vimos como fazer as principais ações usando o Git e Github, a partir deste ponto podemos executar um grupo de comandos que consiste na grande vantagem de se usar o Github: a colaboração com controle de versão. Abaixo um resumo dos comandos abordados:

Comando Ação
fork Copia um diretório do Github
branch Cria um novo “ramo” em seu repositório
pull request Solitação de contribuição
merge Incorporar um ramo criado ao ramo “master”



Para colaborar em um repositório você antes de mais nada copiar os arquivos para o seu Github. Essa ação é executada com o comando fork. Por exemplo, se dirija ao respositório de teste - ciencia_colab_a4 e localize no alto da página o botão fork. Depois escolha onde você gostaria de colocar, escolha seu usuário.
